# Чеклист: подготовка сценарного контроля

## До начала работы

- [ ] Определена дисциплина и тема контроля
- [ ] Сформулированы проверяемые результаты обучения (2-4 результата)
- [ ] Выбрана профессиональная роль студента в сценарии (например: менеджер проекта, инженер, аналитик, консультант)
- [ ] Есть доступ к курсу Moodle с правами редактирования
- [ ] Убедиться, что модуль H5P активирован в Moodle (если пункт «Интерактивный контент» не отображается, обратиться к администратору)
- [ ] Скачан файл шаблона (templates/шаблон-сценарий.h5p)

## Подготовка контента

### С помощью AI (рекомендуется)

- [ ] Сгенерирован кейс с помощью Промпта 1 (см. docs/промпты.md)
- [ ] Сгенерированы точки принятия решений с помощью Промпта 2
- [ ] Сгенерирована обратная связь с помощью Промпта 3
- [ ] Сгенерированы информационные экраны с помощью Промпта 4
- [ ] Все сгенерированные тексты проверены на предметную корректность

### Вручную

- [ ] Написан текст кейса (профессиональная ситуация, 3-5 предложений)
- [ ] Подготовлены точки принятия решений:
  - [ ] Решение 1: вопрос + 3-4 варианта ответа
  - [ ] Решение 2 (усложнённый путь, после ошибки в Решении 1): вопрос + 3-4 варианта
  - [ ] Решение 2 (благоприятный путь, после верного Решения 1): вопрос + 3-4 варианта
  - [ ] Решение 3: вопрос + 3-4 варианта ответа
- [ ] Для каждого варианта ответа написана обратная связь (2-3 предложения)
- [ ] Подготовлены два варианта развития ситуации после Решения 1:
  - [ ] Усложнение (последствия ошибки)
  - [ ] Благоприятное развитие (результат верного решения)
- [ ] Подготовлен текст приветственного и итогового экрана

## Сборка в Moodle

- [ ] Шаблон H5P импортирован в курс (Шаг 2 из инструкции)
- [ ] Все placeholder-тексты заменены на подготовленный контент (Шаг 3)
- [ ] Ветвления настроены корректно (каждый вариант ведёт к нужному экрану)
- [ ] Настроены параметры задания: сроки, попытки, оценивание (Шаг 4)

## Проверка перед публикацией

- [ ] Сценарий пройден по «верному» пути (все ответы правильные)
- [ ] Сценарий пройден по «усложнённому» пути (ошибка в Решении 1 → усложнённое Решение 2)
- [ ] Сценарий пройден по «благоприятному» пути с ошибкой в Решении 2
- [ ] Обратная связь отображается корректно после каждого выбора
- [ ] Результат фиксируется в журнале оценок Moodle
- [ ] Время прохождения адекватное (рекомендуется 20-40 минут)
- [ ] Студенты предупреждены: сценарий нужно проходить целиком за один раз (прогресс не сохраняется)
- [ ] Коллега или ассистент прошёл сценарий как тестовый студент
