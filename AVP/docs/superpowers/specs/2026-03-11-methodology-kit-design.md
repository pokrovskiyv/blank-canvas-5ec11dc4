# Методический комплект для сценарного контроля в Moodle — Спецификация

## Контекст

Заявка на конкурс ТПУ "Лучшая педагогическая практика", номинация "ПРО-технологии". Существующий H5P Branching Scenario апробирован на 22 студентах (Менеджмент, английский язык). Дедлайн: 27 марта 2026.

## Цель

Создать комплект материалов, который:
1. Описывает методическую систему для жюри (README)
2. Позволяет любому преподавателю ТПУ создать свой сценарный контроль за ~1 час (шаблон + инструкция + промпты)

## Deliverables

### 1. README.md (корень проекта)
- **Аудитория:** жюри конкурса
- **Язык:** русский
- **Объём:** 800-1000 слов
- **Содержание:** проблема → решение → состав комплекта → апробация → тиражируемость → AI → автор
- **Тон:** деловой, конкретный

### 2. templates/шаблон-сценарий.h5p
- **Формат:** H5P Branching Scenario (zip с JSON)
- **Основа:** упрощённая версия branching-scenario-416.h5p
- **Структура:** 17 экранов, 3 точки ветвления с расходящимися путями после Решения 1
- **Дизайн:** нейтральная цветовая схема (синие/серые тона), чистая типографика
- **Контент:** placeholder-тексты на русском в формате подсказок
- **Универсальность:** не привязан к дисциплине

### 3. docs/инструкция.md
- **Аудитория:** преподаватель с минимальным уровнем цифровой грамотности (знает Moodle, не знает H5P)
- **Объём:** 1500-2000 слов
- **Структура:** Что такое H5P → Что понадобится → Шаг 1-5 → FAQ
- **Тон:** дружелюбный, пошаговый

### 4. docs/промпты.md
- **Аудитория:** преподаватель без опыта работы с AI
- **Объём:** 1000-1500 слов
- **Содержание:** 4 промпта (кейс, ветвления, обратная связь, инфо-экраны) + советы по доработке
- **Формат промптов:** универсальные, с [КВАДРАТНЫМИ СКОБКАМИ] для подстановки
- **Совместимость:** ChatGPT, Claude, YandexGPT

### 5. docs/чеклист.md
- **Объём:** ~1 страница
- **Формат:** markdown-чеклист с checkbox
- **Секции:** до начала → подготовка контента → сборка в Moodle

## Технические детали H5P-шаблона

H5P — zip-архив со структурой:
- `h5p.json` — метаданные
- `content/content.json` — содержимое сценария
- Библиотеки H5P (BranchingScenario, AdvancedText, CoursePresentation, BranchingQuestion)

Шаблон создаётся путём:
1. Распаковки существующего .h5p
2. Анализа JSON-структуры
3. Создания упрощённой версии с placeholder-контентом
4. Упаковки обратно в .h5p

## Ограничения

- Шаблон не содержит Course Presentation с интерактивными заданиями (drag-and-drop, fill-in-the-blanks) — только текстовые экраны и вопросы с ветвлением. Это упрощает адаптацию.
- Инструкция описывает импорт и адаптацию шаблона, не создание с нуля.
- Промпты универсальные, без специализации под конкретные дисциплины.
